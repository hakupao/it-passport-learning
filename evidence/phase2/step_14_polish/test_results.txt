Test discipline note (Session 46)
================================================================

useFocusTrap.ts is ~115 lines of DOM-side-effect code (document.addEventListener,
focus(), querySelectorAll). The existing project test config is `environment:
"node"` with no jsdom/happy-dom installed.

Options considered:
  (a) Install jsdom or happy-dom as devDep → write hook test
  (b) Refactor hook to take query-fn dependency injection → write pure test
  (c) Defer to Session 47 axe-core integration smoke → no unit test this step

Decision: (c) — matches the existing project discipline:
  - 9 prior steps wrote 0 component tests; all UI behavior verified via Chrome
    DevTools MCP smoke (Session 41-44).
  - useFocusTrap is structurally analogous to historyStore.ts (DOM-side-effect
    consumer); historyStore was tested via injected `Storage` interface, but
    the DOM API surface useFocusTrap touches (focus(), querySelectorAll) is
    larger and a fair stub would mostly re-implement the hook.
  - Session 47 will run MCP `lighthouse_audit` (a11y score ≥95 target) +
    axe-core via `evaluate_script` — both exercise the hook end-to-end.
  - Regression net: tsc strict + ESLint covers compile-time correctness in
    the hook's 8 consuming sites; 277/277 ratifies no logic regression
    elsewhere.

Documented in design_notes.md §4.
