## 市面教科書调研

调研了 6 本主流畅销教材：

| 书名 | 出版社 | 页数 | 顺序 | 核心特色 |
|------|--------|------|------|----------|
| いちばんやさしい | SBクリエイティブ | ~600p | ストラテジ先 | 6年销量第1，彻底讲解 |
| キタミ式イラストIT塾 | 技術評論社 | ~592p | テクノロジ先 | 全页手绘插图，视觉型 |
| かんたん合格 | インプレス | ~480p | テクノロジ先 | 最精简，每节标出题数 |
| かやのき先生 | 技術評論社 | ~496p | テクノロジ先 | 「○○といえば××」联想法 |
| 出るとこだけ | 翔泳社 | ~432p | AI/新技術先 | 极简，只收高频内容 |
| 勝者のITパスポート | TAC出版 | ~580p | テクノロジ先 | 600+插图，WebApp三合一 |

### 关键发现

1. **4/6 本选择テクノロジ优先** — テクノロジ出题占 45%（最高），且技术概念有实物可图解
2. **所有书都配 Web 练习应用** — 单书已不够，数字化练习是标配
3. **即时反馈是共识** — 每节后立刻做历年真题，不攒到章末
4. **联想记忆 > 死背定义** — 「○○といえば××」方式读者评价最高
5. **出题频率透明化趋势** — 让学生知道"这里会考几题"
6. **顺序不是绝对的** — 关键是解释深度，いちばんやさしい（ストラテジ先）靠 600 页详解一样成功

### シラバス规模

- Ver.6.5 共 ~2,651 个用語
- 零基础建议 100~200 小时
- 新增用语集中在 AI/IoT + セキュリティ

## 决策

### D-114: 学習路径組織方式 — 双軌導航

**内容**: 采用双轨导航设计：
- **主导航**: 按シラバス官方树结构（大分類→中分類→小分類），用户可自由跳转
- **推荐学習路径**: 按 **テクノロジ → マネジメント → ストラテジ** 引导初学者

**理由**:
1. 4/6 本畅销书采用テクノロジ优先 — テクノロジ出题占 45%（最高 ROI），技术概念有实物可图解，适合零基础
2. 交互学习型的优势 — 印刷书只能选一种顺序，Web App 可以同时提供官方树 + 推荐路径
3. テクノロジ的题目更适合交互练习（Quiz + Tutor），ストラテジ的法务/财务概念反而更抽象

**拒绝的方案**:
- 纯シラバス顺序（ストラテジ先）— 法务/财务对零基础更抽象，且没有 600 页篇幅详解弥补
- 纯出题频率排序 — 打散知識逻辑关联，不利于理解式学习
- 完全自由无引导 — 零基础需要推荐路径，不能只给一棵树

## 章节内容结构讨论

用户回答:
1. **内容粒度**: 每个用語都详细讲（全覆盖，像「いちばんやさしい」600 页策略）
2. **交互联动**: 深度嵌入 — 页面内直接嵌入過去問即时练习，答错自动调出 AI Tutor
3. **时间预算**: 每学习单元 ~15 分钟

### D-115: 学習ユニット内容架构

学習ユニット（5~8 用語 / ~15 分钟）为 Stage 4 原子生成单位，四段结构：
1. ユニット概要（出題頻度バッジ + 预计时间）
2. 用語講解 × N（定義 + 解説 + 例え + 記憶フック + 図解 + 即時チェック）
3. ユニットまとめ（記憶フック一覧 + 正答率）
4. チャレンジ問題（3-5 道混合難度過去問）

数字验证: ~2,651 用語 / 5-8 per unit = ~330-530 ユニット × 15 min = ~82-132 小时（符合市面建议 100-200 小时）

### D-116: 記憶フック「○○といえば××」为标准配置

每个用語必须生成一条联想钩子。

### D-117: ユニット内用語排列 — 概念依赖优先 + 出題頻度辅助

- 主排序：概念依赖（前置概念排前面）
- 同级并列时：出題頻度高→低
- 拒绝: 纯シラバス顺序（参考用非教学用）/ 纯出題頻度（破坏理解链）/ 随机（无教学价值）

---

### D-118: Stage 4 输出 JSON Schema

- `unit_index.json` — 全局索引 + 推荐学习路径 (`learning_path[]`)
- `units/{id}.json` — 单ユニット完整内容（overview + terms[] + summary + challenge_questions[]）
- Unit ID 格式: `{category}-{major_id}{minor_id}-u{nn}`（例: `tech-0701-u01`）
- Quiz 引用而非内嵌: `inline_quiz` / `challenge_questions` 存 question_bank ID
- 三语平铺: `_jp` / `_zh` / `_en` 后缀，不嵌套
- `figure` 字段可选（~30% 用語需要）
- `prerequisites` 在 index 层管理
