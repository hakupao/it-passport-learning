### 第2の発見 — 「stored 開示=エコー / 非開示=ハルシネーション」→ 真因は MODEL ✅

D-123 の改良 scan（印刷文を先に逐語転写）を 2015h27h で再検証 → **エコー継続**（q10/q11 で agent が `printed_stem` に stored を丸写し → 偽陰性）。
→ ブラインド転写（stored 非開示）に切替 → 今度は **ハルシネーション**（q001 clean を「情報セキュリティ…」と捏造、q004「SNS導入」を「DX戦略」と捏造）。
→ ページ画像 1432×2026(≈173dpi)、Claude vision は ~1.15MP にダウンサンプル → dense 日本語が潰れる仮説。だが**主ループ(Opus)は同じ画像を Read して q10/q11/q85 を正確に読めていた**。

**決定的切り分けテスト**（Opus agent、ブラインド、page-05）:
- Opus + 既存173dpi フルページ → q10/q11 を**完璧に逐語転写**（q10=「ソフトウェアライフサイクル…要件定義プロセス」, q11=「部品Z歩留り…原材料投入量kg」225/250/900/1,000）。
- Opus + 300dpi クロップ → 同じく完璧。
→ **真因は MODEL**: デフォルト `explore` は小モデルで dense 日本語 OCR 不可（エコー/ハルシネーション）。`model:'opus'` なら既存画像でもブラインド正確転写。再レンダ不要（複雑な表のみ hi-dpi クロップ補助）。
