### Step 1 — Pre-processing (script)
- Build `page_figure_map.json`: group 535 figures by page

### Step 2 — Vision extraction (grid overlay method)
- **Attempt 1 (failed)**: Raw pixel coordinate estimation — vision model returned wildly inaccurate bboxes (captured lesson headers instead of figures)
- **Attempt 2 (adopted)**: Grid overlay method — draw 10×14 red grid on each page, model reports grid cell ranges instead of pixels. Much more accurate.
- Grid: 10 cols × 14 rows = ~144×146 px per cell
- Pilot ch02 (27 figs, 20 pages) validated quality
- Full batch: 10 parallel background agents, 1 per chapter group
- All 535 figures identified successfully

### Step 3 — Crop & convert (Pillow script)
- Convert grid cells → pixel coordinates → PIL crop → WebP quality 85
- Ref names with "/" or duplicates sanitized with `p{page}_` prefix
- 535/535 crops successful, 0 errors

### Step 4 — Manifest & validation
- `figure_manifest.json` with all metadata + paths + dimensions
- Rule A audit N=5, independent reviewer

## 6. Issues Discovered & Method Pivot

### Grid overlay method (attempt 2) — quality insufficient

User 广泛抽检后发现 grid 方法精度不足：
- **小图标（icon-small, mascot-icon）**: ~30% 正确率，grid 精度不够定位
- **中大型图**: ~95% 内容正确，但 ±146px 导致多截正文/截断边缘
- **用户评价**: "不够精确"

Confirmed errors fixed manually: 13-04-fig06 (p463), 12-02-fig03 (p424), p210_img-0 (p210), fig-08-05-03 (p311)

### OpenCV 边框检测 (attempt 3) — user approved ✅

User 观察到关键规律：**有用的图几乎都有边框**（黑色实线矩形 or 茶色圆角矩形 or 橙色边框）。

V2 prototype 在 10 页上测试：
- 纯脚本 OpenCV contour detection，**像素级精度**
- 检测黑色(black)、茶色(brown)、橙色(orange)、灰色(gray) 四种边框
- 去重逻辑（IoU >30% overlap → 保留大的）
- User 评价: **"特别准确！非常满意"**

### Method comparison

| | Attempt 1 (raw pixel) | Attempt 2 (grid) | Attempt 3 (OpenCV border) |
|---|---|---|---|
| 定位精度 | ±500px (unusable) | ±146px (coarse) | **±0px (pixel-exact)** |
| 依赖 AI | Yes (localization) | Yes (grid cell ID) | **No (pure script)** |
| 小图标 | ❌ | ❌ | N/A (no border → skip) |
| 中大型图 | ❌ | ~95% | **~100%** |
