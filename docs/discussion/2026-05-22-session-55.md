**B.1 design decisions** (in-source LDs LD-Module-B-1~4):

- **LD-Module-B-1** Escalation criterion = user-explicit harder-reasoning
  ask OR retry-after-low-confidence Sonnet response. Detection
  heuristics + UI hint deferred to Module C Step C.2; B.1 just exposes
  the typed `escalate?: boolean` selector slot.
- **LD-Module-B-2** Tutor role is anthropic-pinned in `getModel` —
  `provider` arg + `LLM_PROVIDER` env both ignored on the tutor path.
  Rationale: D-103 §2.4 ephemeral cache is anthropic-specific.
- **LD-Module-B-3** Dual API — `getTutorModel(opts)` ergonomic
  escalation selector + `getModel("tutor")` matrix-style accessor.
  Both converge on `getTutorModel()` internally so the selection
  logic stays single-source.
- **LD-Module-B-4** `DeepseekRole = Exclude<ModelRole, "tutor">` typed
  record key — keeps `DEEPSEEK_MODEL_BY_ROLE` from gaining a meaningless
  `tutor: "..."` placeholder.

**B.2 design decisions** (in-source LDs LD-Module-B-5~9):

- **LD-Module-B-5** Two-marker nested-breakpoint cache layout (vs
  single-marker). Outer = `TUTOR_SYSTEM_INSTRUCTION` invariant across
  users; inner = `formatTutorPreamble(ctx)` per-session. When user
  mutates progress mid-session, inner breakpoint invalidates but outer
  still hits — partial cache-hit recovery improves D-103 §2.4 ≥80% target.
- **LD-Module-B-6** SYSTEM ~150 tokens, 5 paragraphs, ≤300-token reply
  budget. Locked text minimal because it's the byte-stable cache key.
- **LD-Module-B-7** Deterministic markdown-heading preamble with
  `(count)` summaries + `(none)` empty markers — consistent shape across
  cold/warm states = cache-friendly.
- **LD-Module-B-8** Builder accepts `ModelMessage[]` (post-convert) for
  pure node-env testability; UIMessage→ModelMessage conversion is the
  future `/api/tutor` route's job (mirrors Phase 2 chat.ts posture).
- **LD-Module-B-9** Vitest **inline-snapshot** locks SYSTEM byte content
  — drift triggers test failure (Rule D reviewer surveillance for the
  cache key).
