## 9. Turn 9 — first AI SDK type error (LD-Module-B-15-A fix)

Initial `pnpm exec tsc --noEmit` after wiring providerOptions into the
`streamText` calls produced 5 errors (one per route + tutor):

```
Type 'Record<string, unknown>' is not assignable to type
  'SharedV3ProviderOptions'.
  'string' index signatures are incompatible.
    Type 'unknown' is not assignable to type 'JSONObject'.
```

AI SDK's `SharedV3ProviderOptions = Record<string, JSONObject>` requires
JSON-serialisable inner values; our helper's return type was
`Record<string, unknown>` which widens to non-JSON-compatible.

**LD-Module-B-15-A applied** — declared local `ProviderOptionsShape =
Record<string, Record<string, JSONValue>>` type alias in provider.ts;
both `getPhase2ProviderOptions` and `getTutorProviderOptions` now return
this shape. JSONValue is re-exported from "ai" (no extra dep needed).
Five tsc errors resolved on first re-run.

Tried importing `ProviderOptions` from `@ai-sdk/provider-utils` first, but
that package is a transitive dep (pnpm) without a direct `package.json`
entry, so the import path failed module resolution. The local alias is
the cleanest path forward — no new dep + JSON-strict shape preserved.
