### 4.1 Q12 (OQ-16) ✅ — `pyproject.toml` 配置细节

#### 议题边界 (per Session 01 §8 预告)

#### User 回答（精简整理）

> "这几个我都不懂，你来做决定，我希望是不偷懒可以增加成本（时间+token），规范性高，易读好维护"

User 明确**授权 Claude 决定**，并设三条优化目标:

1. 不偷懒（即不取最简捷径）
2. 成本可增（时间 + token）
3. 规范性高 + 易读好维护

#### Claude 决策过程

按 user 授权 + 三条目标，Claude 调用 `context7` 拉取 uv + hatchling **官方文档**核对 canonical 写法（不靠记忆），结论：

- **a → hatchling**: PyPA 旗下，uv 默认推荐，被 Pydantic / FastAPI / Anthropic SDK 等主流项目统一使用。setuptools 虽 still works 但 src-layout 配置 verbose；poetry-core 与 poetry 工具链耦合，混搭怪；pdm-backend / flit-core 社区小。规范性最高 = hatchling。锁 **D-036**。
- **b → `>=3.11,<4.0`**: 下限不动 D-024。上限 `<4.0` 是防御性，避免十年后 Python 4.x 出现时被自动解析进来。3.12 / 3.13 没加为下限因为没有特性是当前必需，且要保留新手 / 老 Mac 余量。锁 **D-037**。
- **c → 模式 β 实化版**: 根 + 子双 pyproject。uv 官方 albatross 示例就是这个形态：根 `[project]` 含元数据（虚拟伞包，`Private :: Do Not Upload`）+ `[tool.uv.workspace]` + 全局 dev tools；子 = 完整 cert-extractor 包定义 + hatchling 配置。锁 **D-038**。
- **d → 5 条实施细节细化**: glob members / `workspace = true` / 单根 lock / PEP 735 `[dependency-groups]` / Private classifier。锁 **D-039**。

#### Trade-off 备查（为何不选别的）

| 维度 | 拒绝项 | 拒绝理由 |
