# Session 69 — Stage 2.5 全量 AI 審査実行

**Date**: 2026-05-27

## 完了事項

### 1. PDF 画像変換 (Step 1)

全 29 PDF を pdftoppm で 200 DPI PNG に変換。6 並行処理。

- 29 PDF → **1,450 ページ画像**
- 出力: `data/ip/exams/pages/{exam_id}/page-NN.png`

### 2. 並行 AI 審査 (Step 2-3)

29 套試験を executor agent で PDF 画像 vs JSON 逐題対照。各 agent が全ページを読取り、stem_jp / choices_jp / has_figure / figure_description を検証。

**実行統計:**
- 29 agent (6 並行 × 5 ラウンド)
- 7 套が rate limit / session limit で失敗 → 全数リトライ成功
- 1 agent あたり約 50-100 tool calls, 5-12 分

**審査結果:**

| 指標 | 数値 |
|------|------|
| 審査題数 | 3,038 題次 |
| PASS (無修正) | 1,784 題 (58.7%) |
| FIX (要修正) | 1,194 題 (39.3%) |
| MISSING (補録) | 60 題 (2.0%) |
| 図表更新 | 301 題 |

**系統的問題パターン:**
- 数字 0↔9 誤認識 (最多): 0.9→9.9, 100→199, ISO 14001→14991
- 選択肢入替え: 図表題欠落による隣接題のシフト
- Q99/Q100 合併: 全年度で Q100 題幹が Q99 選択肢末尾に混入
- Q1 表紙汚染: 全年度で注意事項テキストが題幹に混入
- has_figure 漏れ: 表/図/チャート含有題が false 標記
- 漢字/仮名誤認: バ↔パ, 把握→把掘, 稼働→乏働
