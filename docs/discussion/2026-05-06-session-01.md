### 1.2 真实动机（关键）

> User 原话（精简整理）：
>
> 「我想学习并通过 ITパスポート 考试。但日语对我比较难——技术名词的概念我都懂（CPU、TCP/IP、ROI 这些），但读不懂日语片假名/平假名的字形，这是阻碍。一旦理解一次了，后续我就记住了。同时我意识到非母语者都会遇到这个问题，所以我想抽出资源做成多语（中+英+日）。然后做（A）个人备考工具、（B）题库网站、（C）AI 学习助手。如果这次跑通，其他资格类的学习只要有资源都可以这么来做。」

**核心 insight**：「**非母语技术学习者卡在假名识读，而不是概念理解**」——这是整个架构的载重动机。每一个设计选择都应该服务于这个 user pain。

## 5. Phase 1 架构（已锁部分）

```
EPUB / PDF / TXT / HTML / ...  ← Phase 1 必须多源（待 Topic #1 细化）
   ↓ Source Reader (pluggable, OQ-08/OQ-09)
原始资产（图片 / 文本 / 元数据）
   ↓ OCR Engine (pluggable; v1 主力 = Mistral)
逐页 Markdown
   ↓ Page Classifier (Claude)
带标签的页面（cover / TOC / 正文 / 術語 / 例題 / 練習 / 過去問 / 章末まとめ / 空页）
   ↓ Hard-page Re-OCR (Claude Sonnet 4.6 Vision, 条件触发)
清洗后逐页 Markdown
   ↓ Structuring (Claude)
实体：chapters / sections / terms / questions / tables / figures
   ↓ Glossary 抽取（在翻译之前！）
锁定 terms_glossary.json（user 抽检）
   ↓ Trilingual Translation (Claude, glossary-constrained)
三语实体
   ↓ Audit (per 规则 A)
   ↓ 多格式导出 (pluggable)
output/itpassport.json + jsonl/ + markdown/ + itpassport.db
```

## 7. 工程纪律（Tier 3，per user CLAUDE.md）

- **规则 A（语义抽检强制）**：压缩率/改写率 >50% 的步骤必须做 N 样本独立抽检；PASS 需 writer-PASS + reviewer-PASS + 独立样本 PASS；结构 PASS ≠ 业务 PASS
- **规则 B（失败归档不删）**：所有失败 attempt 归档到 `failures/step_NN_attempt_X.md`，含输入/产物/技术判定/业务判定/下一 attempt 输入
- **规则 C（Retro 强制）**：Phase 1 收尾前必写 RETROSPECTIVE.md，至少三段：保留下来的做法 / 必须补上的缺口 / 关键决策复盘
- **规则 D（审阅隔离）**：Writer agent 与 Reviewer agent 不能是同一 subagent_type；同 context 自审 = 无审
