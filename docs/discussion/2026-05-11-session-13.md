> Locked Session 13, 2026-05-11. ADR: `docs/decisions/D-079-step611-cadence.md`.

**One-line spec**: 5 user-confirmation gates at high-risk stage boundaries (Stages 1 / 4 / 4.5 / 5 / 6); mechanical stages (0 / 2 / 3) auto-pass. Each gate emits a checkpoint JSON, halts runner, requires user "go" to resume. Idempotent resume via `stage --from N` subcommand.

**5 gates** (full table in ADR §2.1):

| Gate | After Stage | Halt criteria | Reason |
|---|---|---|---|
| ① | 1 OCR | page count + cost.json within ±10% | catches Mistral wholesale failure |

**Why these 5**: Plan-B history (Session 09b) showed the bug class "Stage 4 answer_index drift" + "Stage 5 jp-mutation" only surfaced at Stage 6 audit verdict; at 579-page scale, "discover at end → re-run from N" is disproportionately costly. **Rejected alternatives** (full list in ADR §3):

### Behavioral choice (documented deviation from D-080 §2.1 literal reading)

D-080 §2.1 says "splitter logs a WARN... and emits both original AND split
entries" for the unbalanced case. Literal implementation would mean synthesizing
partial split entries when jp/zh/en disagree on count — but the synthesized
entries can only be lossy (pad with empty strings → Trilingual validator skip)
or non-deterministic (pair by index → silently aligns wrong concepts).

Implemented behavior: in the unbalanced case, **keep original only + record
warn**. The warn carries diagnostic counts so 6.11.A.4 evidence can surface
exactly which entries need human curation. Stage 6 D13 detector will still
fire on the (preserved) original entry, so the polish target "D13 INFO = 0"
is achieved only for balanced cases — unbalanced cases need human review,
which is the right outcome per D-080's "never silently drops data" spirit.

This is a refinement of D-080 §2.1, not a contradiction; documented here
rather than as a new D since the divergence is small + the ADR's intent
("don't silently drop") is preserved.
