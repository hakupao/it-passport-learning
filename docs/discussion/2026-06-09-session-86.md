## 4. 実装前の重要発見 (Q2 の精緻化)

### Discovery 1 — raw bank は内部 cruft + 破損 OCR backup を抱える
`question_bank.json` は 1 問あたり ~40 field、大半がパイプライン provenance/audit:
`stem_jp_corrupted_backup`(×251) / `choices_jp_corrupted_backup`(×174) / `s027_severity` / `*_resourced_s7x` / `figure_repaired_s73` / `s027_unresolved` 等。
→ raw を**そのまま** un-gitignore すると、破損 OCR backup + 修復トレイル全部を公開 git へ晒す。
→ **精緻化**: raw bank は gitignored 維持 (source material、exams/ と同格)、**クリーン派生 corpus `data/ip/quiz/`** を生成して un-gitignore する。同じ結果 (IPA データが公開 repo に載る=ユーザー Q2 の意図) を、クリーンに達成。教科書が派生 corpus で raw syllabus は gitignored、という S85 パターンと一致。翻訳/解析の自然な置き場でもある。

### Discovery 2 — figures は 109M
`figure_path` → `figures/` (467 参照 PNG, 計 **109M**)。`source.page_image` は `pages/` (762M, full page、quiz 不要)。
→ 109M を公開 repo へ = 実 repo 肥大 (D-133 の容量顧慮)。**精緻化**: 参照される 467 枚のみ、PNG ロスレス最適化 (oxipng 等) して派生 corpus に同梱。commit 前に実測。

### Discovery 3 — 一部 figure 問の stem に figure-OCR garble
例 q012 stem に `| @ | [ 生計画定 - ア...` (図が stem に OCR 混入)。図は `figure_path` で別途描画するため、**projection 段で figure 問の stem から garble を除去**する必要。
