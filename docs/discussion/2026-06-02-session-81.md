### 技術的決定 (Claude 判断、本ログ記録)

- **データ供給**: gitignore された `data/ip/textbook/` (+ source PNG/question_bank は `data/ip/exams/`) から **server 側 fs で直読** (`TEXTBOOK_DATA_ROOT` env、既定 `../../data/ip`、既存 `DATA_PATH` パターン踏襲)。`public/` へのコピーはしない。**commit 対象は harness コードのみ、data は gitignore 維持** (Release モデル不変)。
- **図レンダ**: 生成 SVG は server 側で読んで inline (`dangerouslySetInnerHTML`)。原裁剪 PNG は base64 data-URI で inline。いずれも `public/` 不要。
- **route**: `/[locale]/textbook` (index + schema report) + `/[locale]/textbook/[unitId]` (unit detail)。`force-dynamic` (毎回 fs から fresh 読込)。ThemedPage/shell は使わず plain markup。
- **schema 検証の本体** = TS 厳格型へのデシリアライズ + ランタイム `validateUnit()` (三語完整性 / summary 配列長一致 / 図ファイル解決 / quiz ID 解決)。「JSON が型に乗るか」自体が step3 の核心検証。
