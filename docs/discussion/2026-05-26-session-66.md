## 提取方法の変遷

### 方法1: Claude vision agent (初回試行)

6 並列 executor agent で PDF → JSON 提取を試みたが、1M context credits 不足で大半が失敗。
- Batch 1 (FY2009-2010): 成功 — 4回 × 100題 = 400題
- Batch 2-6: API credit エラーで失敗

### 方法2: Anthropic API スクリプト

`scripts/extract-questions.mjs` を作成したが、API credit 残高不足で実行不可。

### 方法3: Tesseract OCR + テキストパーサ (最終採用)

`scripts/ocr-extract-questions.mjs`:
1. `pdftoppm` で PDF → PNG (300 DPI)
2. `tesseract` (jpn) で OCR
3. 正規表現パーサで問題番号・題幹・選択肢を構造化
4. `answer_keys.json` (pdftotext で 100% 正確に抽出済み) とマージ

パーサ改善 (v2):
- 「間」「開」「関」等の OCR 誤認パターン追加
- セクションヘッダ (「問1から問35までは...」) の除外
- ページ番号ノイズの除去
- 選択肢4パターン対応 (縦並び / 横一列 / 2列 / 単行)

### 解答提取

`pdftotext` による解答例 PDF の直接テキスト抽出 → `answer_keys.json`:
- 29回 × 100問 = 2,900 解答、全数正確に抽出 (100%)

### 判明した品質課題

1. **空選択肢** (~32%): 2列レイアウト・表形式の選択肢が OCR パーサで未抽出
2. **欠落問題** (~8%): 図表を含む問題が OCR 処理で消失
3. **OCR 漢字誤り**: 系統的な字形類似による誤認 (把握→所握, 乖離→乗離 等)
4. **題幹汚染**: 一部の題幹に次の問題テキストや試験説明文が混入

## 技術的教訓

1. **IPA 試験 PDF は画像ベース** (テキストレイヤーなし) — pdftotext は解答 PDF にのみ有効
2. **Tesseract OCR の日本語品質**: 本文認識は概ね良好だが、2列レイアウトの選択肢・表・図の認識が弱い
3. **Claude vision が本来の正解ルート**: PLAN 通り Claude vision で提取すべきだった。API credit 枯渇が根本原因
4. **ハイブリッドアプローチが有効**: OCR で 92% カバレッジ → Claude vision で残り 8% + 品質修正
