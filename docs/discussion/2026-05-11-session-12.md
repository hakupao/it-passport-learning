## 2. Decisions Locked

### D-078 — Stage 7 export v1 design

> Locked Session 12, 2026-05-11. ADR: `docs/decisions/D-078-stage7-export-v1.md`.

**One-line spec**:

Per-page JSON + Markdown export with sidecar `polish_items.json`, ASCII
`[JP]/[ZH]/[EN]` language markers, dual release gate (full D1-D13 Phase-1
re-run + Stage 7-specific contract self-check). v1 scope = JSON + Markdown
only (JSONL + SQLite deferred to v2). D-008 amended.

**8 key design choices**:

1. v1 formats: **JSON + Markdown** (Q1 user answer; defer JSONL/SQLite)
2. File granularity: **per-page** (`pages/page_NNN.json` + `pages/page_NNN.md`) plus `index.json` + `glossary.json` (copy) + `polish_items.json` + `README.md` (Q2 user answer)
3. Per-page JSON envelope: `schema_version` + `cert_id` + `run_id` + `stage` + `page` + `exported_at` + `stage6_verdict` + `leaf_count` + `entities` + optional `polish_items_ref`
4. polish_items.json: **sidecar** with WARN + INFO only (FAIL impossible post-Gate-A); Stage6Issue subset (drops verbose evidence blob) (Q3 user answer)
5. Markdown layout: **language-stacked sections** with `[JP]/[ZH]/[EN]` ASCII markers (user 2nd-round override; emoji flags rejected)
6. Gate A: **full D1-D13** Phase-1 deterministic re-run (user 2nd-round override; "only safety subset" rejected)
7. Gate B: Stage 7-specific contract self-check (5 invariants per ADR §2.6)
8. CLI: `export-trilingual --formats json,md --confirm` mirrors `audit-trilingual` shape

**Rejected alternatives** (full list in ADR §3):

- 4-format v1 — deferred non-JSON/MD to v2 (Q1)
- Single mega-bundle JSON — Q2 per-page
- Bare `list[entity]` no envelope — release artifact must self-describe
- Embedded `quality_flags` per entity — Q3 sidecar
- 3-column parallel Markdown — narrow screen overflow risk
- Emoji flag markers — user 2nd-round override
- Skip Gate A — Q4 dual-gate
- Gate A "only safety subset" — user 2nd-round override (full D1-D13)
- Trust Stage 6 closure without re-check — same reason
- Commit output to git — D-046/D-050 already lock to Release tag
