## 4. Execution log — Step 6.11.E.2 / E.3 / E.4

### 4.1 Build the invocation script (`run_release_publish.py`)

`release.publish()` is a Python API (not a CLI subcommand). Built
a small invocation script under `evidence/` that:

- Loads `index.json`, `polish_items.json`, `glossary.json`,
  `cost.json` from `data/.../output/`
- Adapts shapes for `compose_notes()` expectations:
  - `index_for_notes`: flattens `totals.{pages,entities,leaves}`
    + `glossary_term_count` derived from `glossary.json` length
  - `polish_items`: flattens `{by_page, run_level}` into a single
    list with `category` alias of `issue_type` per `_aggregate_polish`
  - `cost_for_notes`: maps `cost.current.{mistral_usd,anthropic_usd}`
    to flat `mistral_usd_billed` / `anthropic_usd_shadow` keys; sets
    `anthropic_usd_billed=0.0` per D-069 max-plan OAuth
- Builds `GitContext` with current HEAD SHA + Python version + 9
  most-cited ADR ids + run_id
- Two modes: `--dry-run` (stages only, no `gh` call) vs `--confirm`
  (stages + invokes `gh release create` + `gh release view`)

The shape-adapter logic is invocation-side rather than a
`release/notes.py` code change — keeps the v1.0.0 release artifact
free of last-mile patches; the adapter belongs to the orchestrator
glue layer for the specific cert+run pair.
