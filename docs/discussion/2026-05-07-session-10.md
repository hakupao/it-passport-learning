## 2. Decisions Locked

### D-077 — Stage 6 audit reviewer LLM design (two-pass + two-tier verdict + repair_stage)

> Locked Session 10, 2026-05-07 under D-019 §"你来定". ADR:
> `docs/decisions/D-077-stage6-audit-reviewer.md`.

**Architecture summary**:

- **Two-pass**: Phase 1 deterministic Python detectors (D1-D13, 13 invariant checks); Phase 2 LLM reviewer (opus, sub-batched chunk=4, tool_use forced).
- **Two-tier verdict** per page: `translation_fidelity_verdict` (jp→zh/en faithful) + `learner_data_verdict` (data shippable to learners). Composed into `overall_verdict` via FAIL > WARN > PASS + safety-field override.
- **`repair_stage`** tagging on every issue (4 / 4.5 / 5 / 7) via deterministic `REPAIR_STAGE_BY_ISSUE_TYPE` table. LLM emits closed-enum `issue_type` only; Python looks up `repair_stage`. No LLM-emitted routing metadata.
- **Reviewer model**: opus (`claude-opus-4-7`), `tool_choice` forced to single `submit_review` tool, `temperature=0`. Override of D-061 default sonnet, recorded per D-061 §2.3.
- **D-063 retry semantics amended** (§2.6 of D-077): when Stage 6 finds FAIL with non-Stage-6 `most_severe_repair_stage`, the retry target is the upstream stage, not Stage 6 itself. Stage 6 does **not** auto-trigger upstream reruns; user authorizes each.
- **Dry-run plan**: Stage A 5 pages (014 clean / 043 questions / 045 term-heavy + COP21 / 030 hand-translation / 038 hand-translation) → halt-collect + user retro → Stage B 40 pages → halt-collect + user retro.
- **Halt strategy**: collect all issues per page, halt at run end; safety-field FAIL halts Stage A early (after current page). Auto-retry only on transient JSON/API errors per D-063 default `max_retries=3`.
- **Budget for dry-run**: Stage 6 caps raised to $999/$999 shadow (per-stage YAML override per D-071 §2.4) — quality-over-cost feedback. Mistral real-billed cap unchanged.

**Rejected alternatives** (full list in ADR §4):

- Q1 (a) Whole-page LLM-only — rejected; LLM can't be trusted with invariant checks (Plan-B evidence).
- Q1 (b) Two-pass LLM — rejected; redundant against Phase 1 deterministic.
- Q1 (c) Single-prompt structured output — rejected; long-context decay re-enters per D-074 / batch-cap memory.
- Q2 (β) Single verdict + dimension tag — rejected; loses page-level fidelity vs data axis distinction.
- Q2 (γ) Three-tier explicit — partially accepted; `overall_verdict` derived not LLM-emitted.
- Q3 (i) LLM-emitted repair_stage — rejected; routing metadata can't be LLM-trusted for irreversible reruns.
- Q3 (iv) No repair_stage in schema — rejected; D-063 retry routing undefined without it.
- Q4 (I) Stage A = 1 page — rejected; insufficient precision/recall coverage for prompt validation.
- Q4 (III) Skip Stage A → straight 40 — rejected; D-073 sample-first contract.
- Q4 (P) Halt on first FAIL — rejected; user retro needs full failure landscape.
- Q4 (R) Auto-retry on content FAIL — rejected; content FAIL means upstream bug, retrying Stage 6 won't change outcome.
