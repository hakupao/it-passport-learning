Notable shape decisions (LD-Step2-A~H per `evidence/phase3/step_02_triggers/design_notes.md §3`):

| LD | Decision | Why |
|---|---|---|
| **LD-Step2-A** | Chapter scope conveyed via user-message prefix `[Scope: 第NN章「title」 p.A-B] ` (not via SYSTEM mutation) | SYSTEM must stay byte-identical for D-088 §2.3 / D-095 §2.3 stable-prefix cache invariant. Marker rides inside user-message body so DeepSeek prefix cache stays warm across calls. Visible to user (transparent — not a hidden instruction). |
| **LD-Step2-B** | `ParagraphTranslate` uses `useChat()` not custom SSE consumer | /api/chat is already on the AI SDK v6 UI message stream protocol. Reusing useChat = zero new transport. Modal close → component unmount → React's natural cleanup aborts the fetch. |
| **LD-Step2-C** | Separate `<ChapterChatModal />` rather than reusing `<Chat />` | `<Chat />` is bound to D-085 §2.2 global localStorage Resume; chapter chat is single-purpose / ephemeral → mixing scopes would corrupt the global history. |
| **LD-Step2-D** | Selection toolbar scoped via `data-chapter-content="true"` data-attribute DOM walk | selectionchange fires on document outside React tree. Walking up to a sentinel data attribute is the cheapest reliable scope check. |
| **LD-Step2-E** | Selection toolbar = translate-only (no chat/quiz buttons) | LD-2 sub-clarification Session 49 Turn 5: avoids two interaction modes confusion. |
| **LD-Step2-F** | ChapterQuizPicker reuses Phase 2 `<QuizExplain />` verbatim | Zero refactor on Step 10 modal; picker is a thin wrapper supplying QuizSummary. |
| **LD-Step2-G** | `clampTranslateSource` hard caps source at 1000 chars with truncated flag | Avoids token-balloon on accidental "select all" + matches "段落-level" mental model. |
| **LD-Step2-H** | `buildChapterQuestionSummaries` projects off `index.entity_by_id` filtered by page range | Zero new API call; pure projection. |
