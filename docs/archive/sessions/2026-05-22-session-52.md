Notable shape decisions (LD-Step3-A~H per `evidence/phase3/step_03_progress/design_notes.md §3`):

| LD | Decision | Why |
|---|---|---|
| **LD-Step3-A** | progressStore storage key = `itp:book:progress:v1` separate from chat | Independent lifecycles (chat cap=200 messages single-thread per D-085 §2.2; progress per-chapter dict with no cap). Sharing a key would create awkward coupling. |
| **LD-Step3-B** | `markChapterCompleted` IDEMPOTENT — returns same reference on already-completed | Click-spam resilience + safe inside `setProgress((prev) => …)`. Preserves first commit's timestamp. |
| **LD-Step3-C** | Scroll-to-end via IntersectionObserver sentinel sticky-once | Avoids window scroll listener overhead; works regardless of chapter length; sentinel disconnect on first hit so scrolling back up doesn't re-disable button. |
| **LD-Step3-D** | Mount-gate in all 3 client islands | localStorage is browser-only; SSR/initial render can't depend on it. Each island starts with emptyProgress, calls loadProgress in useEffect, gates UI on `mounted=true`. Avoids hydration mismatch. |
| **LD-Step3-E** | BookProgressSummary + ChapterProgressPill as SEPARATE client islands | Keeps `<BookIndex />` itself server-rendered. Cost = 16 localStorage reads per index page; acceptable at α-private scale. |
| **LD-Step3-F** | countCompletedChapters iterates SUPPLIED nns list | Future-proofs against corpus updates (chapter ids could change). Canonical 16-chapter set is authoritative; stale localStorage entries ignored. |
| **LD-Step3-G** | Vercel deploy + Playwright + RETROSPECTIVE in same Step 3 sitting | Phase 3 plans 1 deploy total (vs Phase 2 per-step). Deploy gated to user `push it`; RETROSPECTIVE written assuming deploy gate fires soon. |
| **LD-Step3-H** | recordQuizAnswer SHIPPED-AS-HELPER-ONLY (NOT wired to `<QuizExplain />`) | D-085 §2.4 Phase 2 surface is FROZEN. Modify would re-open a locked surface for non-essential telemetry. Phase 4 wires when adaptive scoring becomes meaningful. |
