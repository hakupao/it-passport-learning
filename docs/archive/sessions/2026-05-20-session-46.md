## 4. Turn 4 — In-step diversions (NOT Rule B archive grade)

### 4.1 tsc strict array-index undefined (3-line fix, ~3 min)

After first `pnpm exec tsc --noEmit` pass:
```
src/lib/a11y/useFocusTrap.ts(76,7): error TS2532: Object is possibly 'undefined'.
src/lib/a11y/useFocusTrap.ts(98,11): error TS18048: 'last' is possibly 'undefined'.
src/lib/a11y/useFocusTrap.ts(103,11): error TS18048: 'first' is possibly 'undefined'.
```

Root cause: project tsconfig has `"strict": true` family which implies
`"noUncheckedIndexedAccess": true` — TS types `focusables[0]` as `HTMLElement
| undefined` even after `focusables.length > 0` check (TS does not narrow
array access by length-precondition).

Fix: replace post-length-check usage with destructured locals + truthy guard.
The pattern is identical to Step 10's `quizScope.ts` (verified during fix).
3 lines net diff in `useFocusTrap.ts`; not Rule B archive grade — this is a
TypeScript strict-mode tooling correction at implementation time, not an
architectural surprise. Documented in `step_14_polish/design_notes.md §6.1`.
