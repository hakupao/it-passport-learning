**Plan-B regression guards (Gate ④)**:
- `_walk_translation(translated, structured)` recursively walks both dicts/
  lists in lockstep. At every dict node with a `jp` key in both, compares
  values — any mismatch flags the page. At every dict node, checks `zh`/`en`

**Tolerance helper**: `_within_tolerance(actual, expected, tolerance)` returns
True iff `actual ∈ [expected*(1-tol), expected*(1+tol)]`. Zero-expected edge
case: only `actual == 0` passes (no fractional band around zero).

### Design notes

**Centralized mapping (D-081 §2.1 contract)**: this module is the **only** place
underscore↔dash mapping happens. `tag_name()` maps forward; `parse_tag()` maps
back; both round-trip losslessly for any valid (cert_id, version) pair.

**Version normalization**: input accepts `1.0.0` or `v1.0.0`; output always
`v`-prefixed. Both `1.0.0` and `v1.0.0` produce the same tag string. Whitespace
tolerated on both args.

**cert_id grammar**: `[a-z][a-z0-9_]*` (lowercase ASCII, starts with letter,
underscores/digits allowed after first char). Multiple underscores all map to
dashes (`aws_ccp_v2` → `aws-ccp-v2-v1.0.0`).

**`ParsedTag` is a `NamedTuple`**: gives `.cert_id` / `.version` field access AND
unpacks as a tuple. Avoided `dataclass(frozen=True)` because there's no behavior
to attach — just a 2-tuple of strings with named slots.

**Tag regex anchored**: `_TAG_RE = ^[a-z][a-z0-9-]*-v\d+\.\d+\.\d+$` — same
grammar as cert_id but with dashes instead of underscores. The trailing `-vX.Y.Z`
is unambiguously the version separator (cert_ids never end in `-v\d+\.\d+\.\d+`).

### Design notes

**`compose_notes` is pure** — takes parsed dicts + raw markdown + GitContext;
returns a Markdown string. Caller (CLI / orchestrator) handles file I/O.
Polish items auto-aggregated by `(category, severity)` and sorted FAIL → WARN
→ INFO → unknown.

**`publish()` 8-step orchestrator** (per D-081 §2.4):
1. Validate output dir (4 top-level files present)
2. Compute tag via `tag_name()` (underscore → dash)
3. Zip output/ → `<cert-dashes>-output-<version>.zip` (recursive, preserves
   per-page nested structure)
4. SHA256SUMS over zip + 4 top-level files (5-line manifest, `sha256sum -c`
   compatible)
5. Compose notes via `compose_notes()` + read intro from `intro_md_path`
6. `gh release create <tag> --title --notes-file --target <sha> <asset>...`
7. `gh release view <tag>` (existence verification)
8. Caller writes evidence markdown using returned `PublishResult` paths.

**`gh_runner` injection** — production default invokes
`subprocess.run([gh ...], check=True, capture_output=True, text=True)`. Tests
inject a `Callable[[list[str]], CompletedProcess[str]]` mock that records
calls + returns a synthetic release URL. Zero subprocess invocation in tests.

**`confirm=False` is fully functional dry-run** — all artifacts (zip, sha,
notes) are written to disk; `gh` is NOT invoked; `release_url=None`. This
makes the entire pipeline locally exercisable before any GitHub state changes.
