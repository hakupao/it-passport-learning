- `evidence/phase2_d089_poc_2026-05-19/measurement.md`（summary，7 sections）

**Key findings**（详见 measurement.md）:

1. **Whole-book footprint**: JSON 798,534 tokens (2.48 MB) / MD 491,154 tokens (1.49 MB)；**FITS 1M ctx at 79.9% / 49.1%** ✅。与 Session 28 β PoC 估计 0.84M tokens 一致（差 5%，token tokenizer 误差内）。
2. **Cache block real**: system (500 est) + glossary 98,410 = **98,910 tokens** vs D-088 §2.3 估计 ~90.5K（+9% drift，small）。
3. **Cache cost projection (Opus 4.7)**: cache write **$1.85**/call (session 首) + cache read **$0.148**/call (后续 5min) + uncached baseline $1.48/call。 与 cost_table.md §2.4 校对一致。
4. **Daily mix corrected**: ~$13.55/day shadow Opus everywhere cached（vs cost_table.md §3 估计 $7.92，+71% 偏高 because 实测 cache block 比估的大）。 对 D-090 三档 candidates 影响 = 建议 soft $3→**$5** / mid $10→**$15** / hard $30 unchanged。
5. **FS+parse latency**: 554 pages JSON parse = **16ms warm mean Mac SSD**；FS read alone 8.5ms warm。 Vercel serverless cold start +50-200ms 但内嵌 FS read 仍 <100ms total = ✅ FS-at-boot 数据源 viable。
6. **Per-scope sample**:
   - chapter (p087-p184, 98 pages): 146,062 tokens = 14.6% ctx
   - single page (p087): 3,017 tokens
   - single entity (question on p087): 501 tokens
   - single glossary term: 78 tokens

**数据源 6 candidates 后过滤**（per measurement.md §4 Implications）:

| Candidate | α-now | β-ready | 评 |
|---|---|---|---|
| (i) FS read at boot | ✅ Excellent | ⚠️ per-instance memory | **主推** for α-now |
| (ii) Vercel Blob | ⚠️ Overkill | ✅ for mutable corpus | β 选项 |
| (iii) KV/Redis | ❌ Overkill | ⚠️ cache layer only | reject |
| (iv) Postgres/Supabase | ❌ Overkill | ⚠️ query patterns | reject |
| (v) Static import bundled | ✅ Possible | ⚠️ bundle +2.48 MB | 次推 |
| (vi) Hybrid (FS + Static) | ⚠️ Premature | ✅ per-scope opt | β option |

**PoC cost actuals**: $0 LLM / $0 billed / wall ~10 min（含 script write + run + json inspect + measurement.md write）。 显著快于 Session 28 β PoC（~30 min）和 Session 29 γ PoC（45 min）。
