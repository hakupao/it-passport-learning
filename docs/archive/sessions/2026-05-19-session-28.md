**β PoC spike result** (corpus token estimation + cost cap math + framework analysis)

PoC 范围：用现有 v1.0.3 canonical output 量 corpus size → 估 token → 算 LLM cost per D-085 §2.4 三档 Chat scope。$0 LLM cost (纯文件分析 + 数学), wall ~5 min.

**Corpus size measurement** (`data/itpassport_r6/runs/dry_run_2026-05-12T13-23-19/output/`):

| Scope | Bytes | Tokens (@3.5 B/token trilingual jp+zh+en+JSON) |
|---|---|---|
| Whole book (`pages/` 554 个 JSON) | 2.48 MB | **~0.74M tokens** |
| Whole book + glossary.json | 2.57 MB | **~0.84M tokens** |
| Per chapter (sample 50 pages) | 247 KB | ~72K tokens |
| Per question-page (page_087) | 9.9 KB | ~2.8K tokens |
| Per single Term / section | ~1-2 KB | ~300-600 tokens |

**Key finding** 🎯: whole-book ≈ **0.84M tokens FITS in 1M context** (Claude Sonnet 4.6 1M / Opus 4.7 1M)。先前 D-085 §5.3 "whole-book scope needs RAG" assumption is **overcautious for α-now scope** — 单一 user, 1M ctx model, single-shot 是可行的。**RAG / chunking 可以延后到 β/γ 多用户时再考虑**（cost scales with concurrent users, not corpus size for α）。

**Cost math** (Anthropic pricing, Claude Sonnet 4.6 baseline: $3/M input, $15/M output):

| Scope | Uncached input cost | With prompt caching (~10% read rate) |
|---|---|---|
| Whole-book Chat (0.84M tokens) | **$2.52/call** | **$0.25/call** |
| Chapter Chat (72K tokens) | **$0.22/call** | **$0.022/call** |
| Question Explain (2.8K tokens) | **$0.008/call** | trivially cheap |

**Daily budget estimate** (α-now 单 user 重用 ~50 chats/day mix):
- 40 question-scoped × $0.008 = **$0.32/day**
- 7 chapter-scoped × $0.022 (cached) = **$0.15/day**
- 3 whole-book × $0.25 (cached) = **$0.75/day**
- **Total cached: ~$1.22/day → ~$35/month**
- **Total uncached: ~$9.42/day → ~$280/month**

→ **Prompt caching 是 cost cap 的关键**。D-090 (Session 29+) 锁 cap 数字时这是基线（推荐 prompt cache 强制 on）。

**Cost cap recommendations**（informs D-090 in Session 29+, not in this session's D-087 lock）：
- 三档 cap 复用 D-071：soft $5/day → mid $20/day → hard $50/day
- Per-query hard cap: $5（即使 whole-book uncached 也 fit）

**Framework analysis** (for D-087 stack lock per D-086 = s1 strict web)

Candidate matrix（informed by D-085 form requirements: streaming Chat + 3 tabs + mobile-ready + Hover popover + prompt caching support）：

| Framework | AI SDK 成熟度 | Prompt cache 支持 | Bundle size | DX | Vercel deploy | 推荐度 (α-now) |
|---|---|---|---|---|---|---|
| **Next.js 15 (React 19) + Vercel AI SDK** | 业界最强 (`@ai-sdk/anthropic` 原生 + streaming hooks) | 原生 `providerOptions.anthropic.cacheControl` | 中等 | 极佳，App Router | 0-config | **★★★★★** |
| SvelteKit + AI SDK Svelte | 中等（`ai/svelte` 支持 streaming）| 通过 AI SDK 间接支持 | 最小 | 极佳，简洁 | 0-config | ★★★★ |
| Astro + React islands | 弱（content-first，islands 模式不适合 streaming Chat 主线）| 间接 | 最小 | 好（content）| 0-config | ★★★ |
| Vite SPA + 独立 Python FastAPI | 自己接 Anthropic SDK；streaming hooks 自己写 | 自己 wire | 最小（FE）+ Python deps | 中（多个 repo / monorepo）| 需自部署 BE | ★★★ |
| Remix / SolidStart / Nuxt | AI SDK 支持都有，但用户基数小 | 间接 | 中等 | 好 | 0-config | ★★ |

**Path-of-least-resistance** 推荐 = **Next.js + Vercel AI SDK + Vercel deploy**：streaming Chat UI 是 Phase 2 关键功能；Vercel AI SDK 是这个域 industry-best；prompt caching 原生支持；Vercel 免费 tier 足够 α-now。

**Contrarian** = SvelteKit + AI SDK Svelte: smaller bundle, cleaner code, 同 deploy 自由；缺点 = 社区小，AI 整合 patterns 少，但都能做到。

- **(a) SSR** (per-request server 渲染; Chat API calls 走 server-side route，secret API key 不暴露)
