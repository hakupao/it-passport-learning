#### Q3 — v1.0.3 `output/` 数据物理 location（apps/web 怎么读到 Phase 1 数据）

D-089 §FsDataSource = "from filesystem"，但 v1.0.3 `output/` 实物在 `data/<cert>/runs/<run>/output/` (D-050 gitignored)，未在 `apps/web/` 里。Step 2 怎么解决？

- **(b)** **symlink `apps/web/data` → `../../data/<cert>/runs/latest/output/`**：开发期 symlink，git ignore；本地 + Vercel 都通过 env var 解析路径；不 copy 数据；但 symlink 跨 OS 行为 quirky，Vercel build 不支持 symlink-to-outside-cwd（需 env 或 build-time inline）。

- **(d)** **暂不解决 / 用 stub data**：Step 2 只 nail interface + FsDataSource impl 用 hand-crafted minimal JSON stub（10 chapter / 50 glossary entry）；真 v1.0.3 数据接入推 Step 3 后端 wiring；最快 unblock interface 设计，但 Step 3 必须补真数据接入。
