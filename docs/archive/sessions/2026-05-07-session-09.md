## 2. 本场决定 / Decisions Locked

### D-074 — Stage 5 system prompt MUST include the "always translate the wrapper" clause

> Locked Session 09, 2026-05-07. ADR: `docs/decisions/D-074-stage5-prompt-wrapper-clause.md`.

**Rule**: Stage 5 system prompt MUST contain a clause explicitly
instructing the model to always return a complete trilingual rendering
of the input, with the glossary lock applying only to *how* locked
surfaces render inside the output, not to *whether* the input is
translated at all.

**Why**: 11 of 393 leaves came back UNTRANSLATED on attempt 001 (opus,
chunk=8). The pattern persisted across chunk size halving (attempt 002,
8 left) and chunk=1 (attempt 003, 1 left). Switching tier to sonnet
*regressed* (attempt 004, 2 left). The fix was the prompt clause +
opus (attempt 006, 0 left). Sub-batching alone is insufficient; the
new prompt clause was the actual root-cause.

**How to apply**: regression-guard test
`test_engine_system_prompt_includes_always_translate_clause` ensures
the clause stays in `TRANSLATE_SYSTEM_PROMPT_TEMPLATE`. Future stage 5
runs (incl. Step 6.11 全本) inherit it automatically.

### 7.7 Lessons (write into Phase 1 retrospective when reached)

- **Rule D writer/reviewer 隔离的实证**：claude self-audit gave PASS for
  Stage 5; user retro caught 2 architectural bugs in <30 min of
  reading. The audit verdict was structurally PASS (everything that
  was checked passed), but **Claude's audit dimensions did not include
  schema invariants like "translated.jp == structured.jp" or
  "question.answer_index against vision_full/ ground truth"**. These
  belong in Stage 6 reviewer's catch list (per user worksheet §D).
- **Glossary as adversarial input**：the longer COP21 zh translation
  in the patched glossary made the wrapper-definition refusal pattern
  *worse* (31 fails vs 11 originally on chunk=8). System prompt size
  + glossary content density correlates with model refusal. Phase-2
  consideration: chunk-size auto-tune based on glossary system prompt
  size.
- **Mistral OCR 答案行 silent loss**：dense multi-column answer lines
  ("問題1-5 ウ　1-6 ウ　…") drop in Mistral OCR for some layouts
  but survive in Vision OCR. Heuristic for Stage 3 trigger needs
  question-page-specific check.
- **Hand-write fallback**：for Phase 1 dry-run, 2/382 hand-written
  leaves (0.5%) is acceptable. For full-book run (~4500 leaves),
  same proportion = ~25 hand-writes. Tooling for "user-author trail
  of hand-edits" should be designed (not yet — Phase-2 OQ).
