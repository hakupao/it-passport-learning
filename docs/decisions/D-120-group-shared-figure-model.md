# D-120 — 連問共有図のスキーマ表現「グループ共有図モデル」新設

**状態**: Locked
**日付**: 2026-05-29 (Session 72)
**文脈**: Phase 5 / Stage 2.6 データ実測審核

---

## 背景

Stage 2.6 Phase B の図表監査で、IT パスポート過去問の**連問（中問/大問）構造**が判明。中問前文ページに置かれた図1/表1 を、複数の sibling 設問（問97〜100 等）が共有参照する。現行スキーマは各設問に `figure_path` が平坦で、**共有図の概念がない**ため:

- sibling 設問が `has_figure=false` のまま図を参照（L4 が「図参照あるが図なし」と検出 = 構造的に良性なのに欠陥に見える）。
- 共有図を各 sibling に複製すると図ファイルが重複し連問構造が失われる。

監査で連問共有図ケースを多数確認（2009h21a / 2009h21h / 2012h24a / 2013h25a / 2013h25h / 2015h27a / 2015h27h / 2011h23tokubetsu 等）。

---

## 決定

**連問グループを一級概念としてスキーマに導入する（グループ共有図モデル）。**

- 連問グループに `group_id`（例: `2012h24a-gC`）を付与。グループに属する設問は `group_id` を持つ。
- 共有図はグループに 1 枚（前文ページから裁剪）。グループのメタ（前文テキスト + 共有図 path/bbox）を保持。
- sibling 設問は図を複製せず `group_id` 経由で共有図を参照。`has_figure` は「この設問が図に依存するか」を表し、共有図参照でも true を許容（参照先は group）。
- Stage 4 教科書生成は group 単位で前文＋共有図＋各 sub-question を扱える。

### 実装方針

- question_bank に `group_id`（任意）+ 別途 `groups` セクション or `data/ip/exams/groups.json`（group_id → {preamble_text, shared_figures[], member_qids[]}）。
- 既存の単問 figure（連問でない固有図）は現行 `figure_path` を維持。
- 具体的なグループ確定・共有図裁剪は Stage 2.6 修復フェーズ or Stage 3 直前で実施（スキーマ確定が先、データ充填が後でも可）。

---

## 却下した代替案

- **各 sibling に figure_path 複製**: 最小実装だが図ファイル重複・連問構造が表現されない。却下。
- **現状維持 + Stage 4 で対応**: has_figure=false の誤検出を恒久的に受容することになり、データ品質ゲートが曖昧になる。却下（ユーザー選択は「グループ共有図モデル新設」）。

---

## 影響

- スキーマ拡張（group_id + groups メタ）。Stage 4 JSON Schema（D-118）に group 概念を追記要。
- Stage 2.6 の figure 欠陥のうち「連問共有図」分はこのモデルで解消。
- 単問の真の figure 欠落（add_figure 群）は本モデルと独立に裁剪追加。
