# D-119 — Stage 2.6 データ実測審核と Stage 3 ゲート

**状態**: Locked
**日付**: 2026-05-29 (Session 71)
**文脈**: Phase 5 / Stage 2 補完完了後、Stage 3 開始前

---

## 背景

Stage 2 は OCR 抽出 → 字符修復 → 全量 AI 審査 → 页映射 → 図表裁剪 → 図表審核と多ラウンド回したが、**各ラウンドは異なる「視点」で異なる欠陥類を暴き続けた**。特に Session 71 の図表審核で偶然 `2011h23a-q026` の **stem 汚染（別問題の設問文が混入）** が発覚 → 「stem×choices×answer×figure の跨字段語義整合性」という視点は**一度も専門に検査されていない**ことが判明。

さらに既存の検証は (a) ほぼ単字段/構造チェック、(b) 抽出と同じ Claude vision で実施＝**同源誤差で互いに見逃す**、という構造的盲点を持つ。答案 key のみ IPA 公式 PDF という外部真相源を持つ。

→ Stage 3（知識マッピング）/ Stage 4（教科書生成）は question_bank を真相源として消費するため、汚染が下流を毒する。**Stage 3 着手前に、未使用の視点で実測審核し、系統的欠陥を除去する**。

---

## 決定

**Stage 2.6「データ実測審核」を新設し、その完了を Stage 3 開始のゲートとする。**

### 1. 審核の徹底度 = 分層（全量 + 抽样迭代）

- **全量センサス**（全 2,900 題、軽量・スクリプト化可能・決定的）: L3 跨题重复/汚染スキャン、L4 図文引用整合性、L6 答案分布異常 + 構造再チェック。
- **抽样深核**（N サンプル + 信頼区間、reasoning/画像で高コスト）: L1 再解答による答案逆検査、L2 跨字段語義整合性、L5 数字/単位完整性、L-ext 外部源交叉核対。
- **反復**: 抽样ラウンドが**新しい欠陥類を出さなくなり**、かつ既知欠陥が修正済みになるまで回す（固定 N ではなく「収束」がゲート）。

### 2. 外部真相源の使用 = 許可（審核用のみ、再分発しない）

IPA 公式/過去問解説站から**サンプル題**を取得し、本地抽出と交叉核対してよい。同源誤差の盲点を破る最強の独立視点。産物には含めず二次分発しない。

### 3. 欠陥への対応 = 確実なものは即時修正 + 不確実は帰档

原 PDF 対照で逐字確認できる確実な欠陥は直接修正（原値を `*_corrupted_backup` 等で温存、`*_corrected_s71/s72` フラグ）。曖昧・要判断のものは欠陥マトリクスに帰档しユーザー裁決を待つ。

### Stage 3 開始ゲート（全て満たすこと）

1. 全量センサス 3 視点のフラグを全件トリアージ（修正 or 意識的受容）。
2. 抽样深核が「新欠陥類ゼロ」のラウンドに収束、かつ critical 欠陥率の実測値 + 95% 信頼区間が記録され、ユーザーが受容。
3. 既知残課題（has_figure 孤児 16 件、Session 68 記載の「題幹-選択肢不整合 ~14 件 @2015h27h/2022r04」、0↔9 数字誤識 ~30 件）が解消。
4. Tier 3 証拠（audit matrix + CI 計算 + Rule A/D 独立審核）が `evidence/phase5/stage_026_*` に揃う。

## 却下した代替案

- **仅統計抽样で準確率を出すだけ**: 全量級の系統的錯位（跨题串題・答案分布偏斜）を見逃す。却下。
- **全量人工級逐題核対（2,900 題）**: 最も徹底だが agent 用量・時間が過大。分層方式で同等の系統カバレッジを低コストで得られるため却下。
- **外部源不使用**: 同源誤差の盲点を破れない。著作権は「審核用・非分発」で問題ないため使用を採用。
